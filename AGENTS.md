# 项目协作规则

## 项目目标

这是一个个人作品集静态网站，用于展示个人介绍、精选项目、技能方向和联系方式，并优先支持 GitHub Pages 部署。

## 技术栈

- HTML / CSS / JavaScript
- 无构建步骤的静态站点
- GitHub Pages 可直接托管

## 运行方式

- 直接打开 `index.html`
- 或在仓库根目录启动本地静态服务：
  - `python -m http.server 8000`

## 验证方式

- 浏览器打开首页，检查首屏、项目区、联系区和移动端布局。
- 修改后至少验证：
  - 桌面宽度
  - 手机宽度
  - 导航锚点
  - 项目数据渲染

## 代码风格

- 保持静态站点简单清晰，避免引入不必要框架。
- 文案和项目数据优先集中在 `scripts/main.js`。
- CSS 使用语义化 class 名，避免过深嵌套。
- 新增视觉资产放在 `assets/`。

## 禁止事项

- 不提交密钥、token、cookie 或任何私密数据。
- 不覆盖用户或其他 Agent 的未提交改动。
- 不强推，不随意执行 `git reset --hard`。
- 不把 GitHub Pages 部署说明写成必须依赖私有服务。
