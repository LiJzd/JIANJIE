# 决策记录

## 2026-06-08

### 决策

首版作品集使用无构建步骤的静态 HTML/CSS/JavaScript。

### 原因

- GitHub Pages 可以直接托管静态文件。
- 项目当前为空，静态实现上线最快，也便于后续其他 Agent 接手。
- 用户尚未提供真实作品数据，先用集中式数据结构降低替换成本。

### 影响

- 不需要 Node.js、Vite 或其他构建链。
- 页面交互保持轻量。
- 适合先上线，再逐步替换真实内容。

### 是否可逆

可逆。后续如果需要复杂路由、CMS 或动画系统，可以迁移到 React/Vite。

## 2026-06-08

### 决策

作品区优先从 GitHub API 读取 `LiJzd` 的公开仓库，并保留静态 fallback 数据。

### 原因

- 用户希望“我的作品都放进去”，公开 GitHub 仓库是当前最准确的作品来源。
- 动态读取可以覆盖后续新增公开仓库。
- fallback 可以避免 API 限流或网络异常时页面空白。

### 影响

- 首次加载作品区会请求 GitHub 公共接口。
- 静态 fallback 需要在重要仓库信息变化时手动同步。

### 是否可逆

可逆。后续可以改成完全静态数据，或引入构建时抓取仓库数据。

## 2026-06-08

### 决策

UI 参考 `LiJzd/ZUOPINGJI` 的视觉风格，但项目展示保持正常作品集表达。

### 原因

- 用户明确要求“只需要模仿风格和效果，项目展示还是按正常的来”。
- `ZUOPINGJI` 的黑红高对比、斜切、硬阴影和半调网点适合作为个人风格。
- 剧情化“劫案/预告信”等文案不适合当前作品集的信息展示目标。

### 影响

- 页面视觉更强烈，更接近参考仓库。
- 导航、项目卡片、弹窗和联系区仍使用项目/技能/关于/联系等常规信息架构。

### 是否可逆

可逆。可以回退到原白底作品集，也可以进一步迁移为 React/Vite 版本以复刻更多动态效果。

## 2026-06-08

### 决策

滚动体验采用 CSS 变量 + IntersectionObserver 实现，而不是引入 React 或动画库。

### 原因

- 当前项目是无构建步骤的静态站点，继续保持部署简单。
- 用户需要强烈滚动动效、反射光束和琴键导航，这些可以用原生 CSS/JS 完成。
- 避免为了动效引入新的构建链，降低 GitHub Pages 部署复杂度。

### 影响

- 背景反射光束会随 `--scroll-progress` 延伸和偏移。
- 区块和项目卡片通过 `scroll-reveal` / `in-view` 类实现入场动画。
- 右侧琴键导航通过 IntersectionObserver 跟踪当前区域，桌面显示、移动端隐藏。

### 是否可逆

可逆。后续如需复刻 `ZUOPINGJI` 的 Shader/WebGL 级背景，可以迁移到 Vite/React 或单独加入 Canvas/WebGL 模块。

## 2026-06-08

### 决策

作品区改为精选展示：继续读取 GitHub 公开仓库用于补充链接、语言和更新时间，但只渲染 `scripts/main.js` 中 `selectedProjects` 指定的代表项目。

### 原因

- 用户明确要求“别展示所有作品，挑选一些就行”。
- 精选展示比完整仓库列表更适合作品集首页，能减少噪音并突出代表方向。
- 保留 GitHub API 补充信息可以避免手动维护链接和更新时间，同时 fallback 仍能离线渲染。

### 影响

- 新增公开仓库不会自动出现在作品区，需要手动加入 `selectedProjects`。
- 作品卡片不再展示星标或 fork 数量，避免把作品价值表达绑定到社交指标。
- 项目封面当前使用仓库内生成的 SVG 视觉资产，后续可替换为真实截图。

### 是否可逆

可逆。若之后需要完整仓库索引页，可以新增单独页面或恢复全部仓库渲染逻辑。

## 2026-06-09

### 决策

为首页增加参考 P3R 官网开场节奏的原创全屏开场动画，使用静态 HTML/CSS/JavaScript 实现，不引入第三方动画库或官网素材。

### 原因

- 用户希望开场有类似 `https://asia.sega.com/p3r/cn/` 的动画体验。
- 当前项目仍以 GitHub Pages 直接托管为优先目标，原生 CSS 动画最符合无构建部署方式。
- 只借鉴蓝黑高对比、斜切切片、短句闪入、倒计时和退场节奏，避免复制官方图像、角色或素材。

### 影响

- 首页加载后会先显示约 4 秒的开场层，用户可点击 SKIP 跳过。
- `prefers-reduced-motion: reduce` 用户不会看到强制开场动画。
- 开场退场计时在页面 `load` 后启动，避免字体或网络请求较慢时过早结束。

### 是否可逆

可逆。可移除 `index.html` 的 `intro-sequence`、`styles/main.css` 的 intro 样式和 `scripts/main.js` 的开场控制逻辑，回到普通首屏。
